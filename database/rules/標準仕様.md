# 標準仕様

過去案件データが参照可能になれば実績ベースに更新する。
現時点は設備制御盤設計の一般的な標準構成・デフォルト値を記載する。

---

## 1. 電源・回路電圧

| 項目 | 標準値 | 備考 |
|---|---|---|
| 主回路電圧 | AC200V 3相 50/60Hz | 国内工場標準。AC400Vは重電設備・大容量時 |
| 制御回路電圧(AC) | AC200V または AC100V | 変圧器(コントロールトランス)で降圧する場合はAC100V |
| 制御回路電圧(DC) | DC24V | PLC入出力・センサ電源の標準 |
| DC電源供給 | スイッチング電源(24V) | パナソニック・オムロン・富士電機等のスイッチング電源を使用 |

---

## 2. 主回路構成（標準）

```
電源 → 主幹ブレーカ(MCCB) → 各分岐ブレーカ(MCCB/MCB)
    → 電磁開閉器(MSまたは電磁接触器+サーマルリレー)
    → モータ / インバータ / サーボドライバ
```

- インバータ使用時: 電磁開閉器ではなく **ブレーカ+電磁接触器**（インバータ内蔵の保護機能を利用）
- サーボ使用時: ブレーカ+電磁接触器（サーボドライバ内蔵の過負荷保護を利用）
- スターデルタ始動が必要な場合: Agent01が確認・フラグを立てる

---

## 3. 保護等級(IP)

| 設置場所 | 標準IP | 備考 |
|---|---|---|
| 屋内・清潔環境 | IP54 | 一般的な制御盤の最低限 |
| 粉塵・切削油環境 | IP65 | 加工機・食品機械等 |
| 屋外設置 | IP65以上 | 雨水・直射日光対策が必要 |
| 防爆エリア | 別途確認 | Agent01が危険場所の有無を確認する |

## 4. 配線色の標準

| 用途 | 標準色 |
|---|---|
| 主回路（三相・単相） | 黒（電線本体） |
| 主回路 絶縁キャップ R/U | 赤 |
| 主回路 絶縁キャップ S/V | 白 |
| 主回路 絶縁キャップ T/W | 黒または青 |
| 接地線(PE) | 緑 |
| AC制御回路 | 黄 |
| DC制御回路(+/-) | 青（極性は端子番号で区別） |
| センサ配線(+/信号/0V) | 茶/黒/青 |

> 客先指定がある場合は客先仕様を優先し、Agent01の仕様解析時に記録する。
> CE対応案件の配線色（IEC 60204-1準拠）は `database/rules/電線選定.md` セクション8を参照。

## 5. 部品メーカーの優先順位

> **注意**: この優先順位は社内デフォルトであり、客先指定・価格・納期・在庫状況により変更される。最終判断は人間が行う。

| カテゴリ | 第1候補 | 第2候補 | 備考 |
|---|---|---|---|
| ブレーカ | 三菱電機 | 富士電機 | 客先指定がなければ三菱電機 |
| 電磁開閉器 | 三菱電機 | 富士電機 | 同上 |
| 電磁接触器 | 三菱電機 | 富士電機 | 同上 |
| インバータ | 三菱電機(FR-A800/E800) | 富士電機(FRENIC) | 既存設備に合わせる |
| サーボ | 三菱電機(MR-J5) | 富士電機(FALDIC) | 既存設備に合わせる |
| PLC | 三菱電機(iQ-F/iQ-R) | — | 客先PLCと通信する場合は客先に合わせる |
| 端子台(単体調達) | Phoenix Contact / IDEC等 | 三菱電機(TB)・富士電機(UK) | 単体調達の場合は他メーカーがコスト有利。接続方式(コネクタ/端子台)は案件の選定出力で判断 |
| センサ | オムロン | キーエンス | 客先指定に従う |
| スイッチング電源 | パナソニック | オムロン | — |

## 6. 標準的な安全回路構成

| 項目 | 標準 | 備考 |
|---|---|---|
| 非常停止回路 | Bコンタクト直列 | IEC 62061 / ISO 13849-1 の適用要否はAgent01が確認 |
| ドアインターロック | リミットSW または安全ドアSW | — |
| 安全カテゴリ目標 | Cat.3 PL d(要件あり時) | CE対応・輸出向けの場合に適用。国内のみは要確認 |

## 7. 容量選定マージン（ブレーカ・電磁接触器）

- モータ負荷: **モータ定格電流の1.25倍以上** を選定基準に（カタログの選定表に従う）
- インバータ二次側: インバータ出力電流に対応したブレーカ・接触器を選定（カタログp参照）
- 余裕率は必ずカタログ記載の選定基準を優先し、本ファイルの値はあくまで目安

## 更新ルール

- 過去案件データへのアクセスが可能になった時点で、実績ベースの標準構成に更新する
