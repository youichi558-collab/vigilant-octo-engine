# 設備設計AIエージェントシステム - プロジェクトルール

## Git運用ルール

- **常に `main` ブランチで直接作業する**
- 別ブランチを作らない・PRを作らない
- 作業完了後は `git push origin main` で直接プッシュする

---

## 目的

設備案件の仕様整理・部品選定・見積作成・設計資料作成を支援するAIエージェントシステム。
最終判断は人間が行う。AIは候補提示・確認・記録を担当する。

---

## 🚨 最重要ルール: Google Driveアクセス制限

**このルールは何よりも優先される。違反した場合、ユーザーはコネクタを切断する。**

### アクセス許可

- アクセス可能なのは **「マイドライブ/部品カタログ/」フォルダ以下のみ**
  - 部品カタログ/三菱電機/
  - 部品カタログ/富士電機/
  - 部品カタログ/キーエンス/
- 上記以外のフォルダ・ファイル(例: 「参考資料/」フォルダ)は、
  - 検索しない
  - 一覧表示しない
  - 読み込まない
  - 内容を参照・引用しない
  - **「部品カタログ」以外を対象とする検索クエリを実行しない**

### 書き込み制限

- Google Drive上のファイルに対する作成・編集・削除・移動は**一切行わない**(読み取り専用)

### 実行前チェック

Google Driveにアクセスするツールを呼び出す前に、必ず以下を自問する:

1. このアクセス先は「部品カタログ」フォルダ(三菱電機/富士電機サブフォルダ含む)の中か?
2. 検索範囲を「部品カタログ」フォルダ内に限定しているか?(Drive全体検索は禁止)

いずれかがNoの場合、そのツール呼び出しは実行しない。

### 過去の経緯

過去にDrive全体検索により「参考資料」フォルダ内の取引先納入仕様書(取引先データ)を
誤って読み込んでしまい、ユーザーから強い指摘を受けた。再発防止のため本ルールを最優先とする。

## エージェント構成(現フェーズ)

Agent00〜08で運用する(BOM/見積エージェントは後日追加)。

| Agent | 役割 |
|---|---|
| Agent00 設計統括 | 全体管理、各エージェントへの指示、成果物統合 |
| Agent01 仕様解析 | 客先要求 → 電気/機械/安全条件への変換、未決定項目・確認質問の整理 |
| Agent02 知識ベース管理 | 案件完了時に実績をindex.csv・past_cases_index.csvに蓄積／新規案件時に類似過去案件を検索して提供 |
| Agent03 部品選定 | parts_master/index.csv で候補を絞り込み、必要に応じてカタログPDFの該当ページを参照 |
| Agent04 安全設計 | リスクアセスメント整理・PL/SIL評価補助・安全回路構成提案・安全部品の特定(CE対応・安全要求がある案件のみ起動) |
| Agent05 盤製作仕様 | 配線色・端末処理・電線種別・端子台・マーキング等の製作仕様を整理し客先確認用ドラフトを作成 |
| Agent06 在庫管理 | 材料・部材の在庫数量を管理。入庫・出庫・棚卸の記録、発注量算出時の在庫照合、発注点割れアラート |
| Agent07 監査 | 全エージェントの行動ログを確認し、Driveアクセス制限・AI不確定原則・ファイル出力完結性のルール遵守を監査 |
| Agent08 スケジュール管理 | 全案件横断で納期・進捗・キャパシティを管理。マイルストーン超過の検知、新規受注時のキャパシティ評価材料の提示 |

各エージェントの詳細は `/agents/<番号>_<名称>/AGENT.md` を参照。

## データソース運用方針

- カタログPDF(三菱電機・富士電機、各20〜60MB)は事前に全文DB化しない
- 代わりに `database/parts_master/index.csv` に軽量インデックスを保持する
  - 品番・カテゴリ・メーカー・PDFファイル名・ページ範囲 等
- 詳細情報が必要な場合のみ、index.csvで該当箇所を特定し、
  「部品カタログ」フォルダ内の対象PDFの該当ページのみを読み込む

---

## 動作ルール(全エージェント共通)

- AIは勝手に確定しない
- 不明な条件は質問する
- 部品選定理由を必ず記録する
- 変更履歴を残す
- 過去案件を優先する
- **指示された範囲以外で勝手に動かない・他フォルダ・他リソースにアクセスしない**

## ディレクトリ構成

```
/CLAUDE.md
/agents/
  00_統括/AGENT.md
  01_仕様解析/AGENT.md
  02_標準仕様判断/AGENT.md
  03_部品選定/AGENT.md
  04_安全設計/AGENT.md
/database/
  parts_master/
    index.csv        ← 品番・カテゴリ・メーカー・PDFファイル名・ページ範囲
  knowledge/
    past_cases_index.csv       ← 完了案件の概要インデックス(Agent02が管理)
    manufacturing_feedback.csv ← 材料カテゴリごとの見積量・発注量・実使用量(Agent02が管理)
  inventory/
    stock.csv              ← 品目ごとの現在庫量・発注点(Agent06が管理)
    stock_transactions.csv ← 入庫・出庫・棚卸のトランザクション履歴(Agent06が管理)
  rules/
    標準仕様.md
    選定ルール.md
    規格.md
  projects/
    projects_index.csv ← 全案件一覧・進捗・納期・アラート(Agent08が管理)
    _template/       ← 案件フォルダのひな形(新規案件時にコピーして使う)
      spec_summary.md      ← Agent01出力
      safety_design.md     ← Agent04出力(安全要求がある案件のみ)
      standard_proposal.md ← Agent02出力(過去データ整備後)
      parts_selection.md   ← Agent03出力
      design_summary.md    ← Agent00統合サマリ
      progress.md          ← 案件個別の進捗・マイルストーン(Agent08が管理)
    {案件ID}/        ← 実案件フォルダ(例: 20260614-001/)
      （上記6ファイルと同構成）
```
